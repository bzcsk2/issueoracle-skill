# IssueOracle Skill 后续阶段实施开发计划

## 0. 总目标

IssueOracle 后续开发不再以“大功能扩张”为主，而是把当前 v0.2 原型打磨成一个可信、可安装、可发布、可被 Agent 稳定调用的 Skill 产品。

核心路线：

```text
v0.2.1 可信发布版
  → 修代码格式、版本统一、CI、Release、pack 去重、eval 严格化

v0.2.2 安装体验版
  → 对齐 last30days-skill 的安装方式、README、NUX、手动安装、构建产物

v0.3.0 经验准入版
  → candidate / reviewed / approved 机制，让 mined experience 不直接污染 review

v0.3.1 GitHub mining 稳定版
  → 限流、重试、缓存、resume、分页、诊断

v0.4.0 Pattern Pack 扩展版
  → 扩到 20-30 个高质量 patterns，重点补 Agent / CLI / TUI 工程 bug

v0.5.0 公开推广版
  → GitHub topics、release notes、demo、docs 精简、贡献流程稳定
```

硬原则：

```text
1. 不做 MCP。
2. 不自动修改代码。
3. 不自动提交 commit / PR。
4. 默认 local-first，不上传用户代码。
5. SKILL.md 是运行时唯一真相源。
6. experience.json 是机器契约，bug-experience.md 只是叙事视图。
7. finding 没有 file/line/pattern/trigger/evidence/boundary 就不能输出。
8. mined pattern 未经 approved 不应进入正式 review。
```

## 1.4 增加 Python 版 skill 打包脚本

### 新增文件

```text
skills/issueoracle/scripts/build_skill.py
```

### 目标

跨平台生成：

```text
dist/issueoracle.skill
```

### 实现要求

1. 使用 Python stdlib。
2. 不依赖 Bash、rsync、tar。
3. 排除 `__pycache__`、`.pytest_cache`、`.DS_Store`、临时文件。
4. 校验 bundle 内必须包含：

   * `SKILL.md`
   * `scripts/issueoracle.py`
   * `scripts/lib/schema.py`
   * `packs/`
   * `references/`
5. 输出 artifact 路径和文件大小。

### 伪代码

```python
from pathlib import Path
import zipfile

SKILL_DIR = Path(__file__).resolve().parents[1]
ROOT = SKILL_DIR.parent.parent
DIST = ROOT / "dist"
OUT = DIST / "issueoracle.skill"

EXCLUDES = {
    "__pycache__",
    ".pytest_cache",
    ".DS_Store",
}

def should_include(path: Path) -> bool:
    return not any(part in EXCLUDES for part in path.parts)

def main() -> int:
    DIST.mkdir(exist_ok=True)
    with zipfile.ZipFile(OUT, "w", zipfile.ZIP_DEFLATED) as zf:
        for path in SKILL_DIR.rglob("*"):
            if path.is_file() and should_include(path):
                zf.write(path, path.relative_to(SKILL_DIR))
    verify_required_entries(OUT)
    print(f"Built {OUT}")
    return 0
```

### 保留旧脚本

`build-skill.sh` 可以保留，但改成薄包装：

```bash
#!/usr/bin/env bash
set -euo pipefail
python3 "$(dirname "$0")/build_skill.py"
```

# Phase 3：v0.3.0 Experience 准入机制

## 3.1 引入 candidate / reviewed / approved 三态

### 背景

`mine` 产生的经验现在不应直接作为正式 review 输入。必须建立准入机制，否则 GitHub issue 噪声会转化为误报。

### 新增目录

```text
~/.issueoracle/
  bugplay/
    candidates/
    reviewed/
    approved/
```

### 新增 schema 字段

修改：

```text
skills/issueoracle/scripts/lib/schema.py
```

给 `BugExperience` 增加：

```python
status: Literal["candidate", "reviewed", "approved", "rejected"] = "candidate"
review_notes: list[str] = []
approved_by: str | None = None
approved_at: str | None = None
```

### 新命令

```bash
issueoracle.py experience list
issueoracle.py experience show <id>
issueoracle.py experience approve <id>
issueoracle.py experience reject <id>
issueoracle.py experience export-approved
```

### 命令行为

```text
mine
  → 默认写入 candidates

experience approve
  → 从 candidates 移到 approved
  → 写入 approved_by / approved_at

review --experience
  → 默认只加载 approved
  → 如果路径是 candidate 文件，必须显式 --include-candidates
```

### review 安全规则

```text
1. 默认不加载 candidate。
2. 加载 candidate 时报告顶部必须显示 warning。
3. candidate finding 的 severity 最高只能是 medium，除非用户加 --trust-candidates。
```

## 3.2 experience.json 成为唯一机器契约

### 修改文件

```text
skills/issueoracle/scripts/lib/experience.py
skills/issueoracle/scripts/issueoracle.py
skills/issueoracle/scripts/lib/render.py
```

### 新规则

```text
1. review --experience path.json → 直接加载 JSON。
2. review --experience bug-experience.md → 自动寻找同目录 experience.json。
3. 找不到 JSON 时才解析 markdown。
4. markdown fallback 必须 degraded warning。
5. markdown fallback 得到的 patterns 默认 status=candidate。
```

### 错误处理

如果用户显式传入 `--experience`，但文件不存在或加载出 0 个 pattern，必须失败退出。

错误示例：

```text
ERROR: --experience was provided but no valid experience patterns were loaded.
Path: ~/.issueoracle/bugplay/bug-experience.md

Expected one of:
- experience.json
- approved/experience.json

Hint:
Run:
  issueoracle.py mine owner/repo
  issueoracle.py experience approve <candidate-id>
```

## 5.2 Pattern schema 支持 typed signals

### 背景

当前 `bad_code_signals` 如果只是字符串，容易关键词误报。

### 新 schema

```yaml
bad_code_signals:
  - kind: required
    value: "allow_credentials=True"
  - kind: required_any
    values:
      - "allow_origins=['*']"
      - 'allow_origins=["*"]'
  - kind: negative
    value: "allow_credentials=False"
  - kind: suppress_if_present
    value: "try:"
```

### 同时支持旧格式

为了兼容旧 pack：

```yaml
bad_code_signals:
  - "allow_credentials=True"
```

加载时转换成：

```yaml
bad_code_signals:
  - kind: required
    value: "allow_credentials=True"
```

### 修改文件

```text
schema.py
pack_loader.py
pattern_match.py
review.py
references/pattern-schema.md
tests/test_schema.py
tests/test_pattern_match.py
tests/test_pack_loader.py
```

### 匹配器规则

```text
required
  → 必须出现

required_any
  → 至少出现一个

negative
  → 出现则降低置信度

suppress_if_present
  → 出现则直接 suppress

optional
  → 出现则加分，不出现不扣分
```
